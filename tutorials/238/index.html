NUI Image Components

Image View는 화면에 Image를 보여줄 때 사용합니다.

이번 튜토리얼에서는 Image View를 어떻게 생성하고 사용할 수 있는지 알려드리고,

주요 기능에 대해서 간단히 설명 드리려고 합니다.

기본적인 Guide와 API 설명에 대해서는 아래 문서를 참고해주시기 바랍니다.

Guide Page   : https://docs.tizen.org/application/dotnet/guides/nui/imageview/

API reference : https://docs.tizen.org/application/dotnet/api/TizenFX/latest/api/Tizen.NUI.BaseComponents.ImageView.html

1. Image View 생성 및 간단한 사용법

Image View는 XAML을 통해 간단하게 생성해서 사용할 수 있습니다.

특히 Tizen NUI XAML Project의 Template을 생성하면 Scene1Page.xaml 생성되며,

Template에서 기본적으로 Image View를 생성하고 있기 때문에 해당 XAML을 수정하여 편하게 사용하실 수 있습니다.

<?xml version="1.0" encoding="UTF-8" ?>
<View x:Class="NUIXAMLProject2.Scene1Page"
        xmlns="http://tizen.org/Tizen.NUI/2018/XAML"
        xmlns:x="http://schemas.microsoft.com/winfx/2009/xaml"
        xmlns:local="clr-namespace:NUIXAMLProject2"
        WidthSpecification="{Static LayoutParamPolicies.MatchParent}"
        HeightSpecification="{Static LayoutParamPolicies.MatchParent}"
        BackgroundColor="White">
    <View.Layout>
        <LinearLayout LinearOrientation="Vertical" LinearAlignment="Center"></LinearLayout>
    </View.Layout>
    <ImageView ResourceUrl="*Resource*/images/tizen.png"/>
    <TextLabel Text="Hello Tizen NUI"
               PixelSize="36"/>
</View>

예를 들면 다음과 같은 형태로 Image View에서 제공하는 다양한 Property들을 통해 다양한 기능을 사용할 수 있습니다.

Property에 대한 설명은 기능을 설명하면서 좀 더 자세히 다뤄보겠습니다.

<ImageView x:Name="imgView"
           ResourceUrl="*Resource*/images/Poster_1.png"
           Size2D="600, 600"
           PixelArea="0.0, 0.0, 1.0, 1.0"
           FittingMode="ShrinkToFit"
           BackgroundColor="#000011"/>

지금까지 아주 간단한 Image View 생성 방식에 대해서 알아보았습니다.

이제부터는 Image View에서 제공하는 세부적인 기능에 대해 설명 드리겠습니다.

2.Image View 기능

Image View는 Image를 화면에 보여줄 때 여러가지 기능을 제공합니다.

이번 튜토리얼에서는 그 중 대표적인 기능 몇가지에 대해서 설명드리겠습니다.

(1) Pixel Area

첫번째는 PixelArea 기능입니다.

해당 기능을 통해 이미지의 특정 영역을 확대해서 보여줄 수 있습니다.

실제 사용 시는 아래처럼 시작점 x,y 영역의 width, height 를 입력으로 받아

Pixel Area 를 설정하고 해당 영역을 zoom해서 표시하게 됩니다.

<ImageView x:Name="imgView"
               ResourceUrl="*Resource*/images/Poster_1.png"
               Size2D="600, 600"
               PixelArea="0.4, 0.4, 0.6, 0.6"
               FittingMode="ShrinkToFit"
               BackgroundColor="#000011"/>

(2) Fitting Mode

다음은 FittingMode 기능입니다.

FittingMode는 이미지 파일을 View에 어떻게 맞출지 결정하는 방법입니다.

크게 아래 4가지 타입을 통해 FittingMode를 설정할 수 있습니다.

(3) Alpha Mask

마지막으로 AlphaMask 기능에 대해 설명드리겠습니다.

Alpha Mask 이미지가 있다면 이를 활용해 이미지에 마스크 효과를 줄 수 있습니다.

마스크 적용은 다음과 같은 방식으로 사용할 수 있습니다.

 <ImageView x:Name="imgView"
              ResourceUrl="*Resource*/images/Poster_1.png"
              AlphaMaskURL="{ResourcePath FilePath=images/mask-large.png}"
              Size2D="400, 400"/>

3. Gif, Lottie 사용법

Gif확장자의 경우도 일반 이미지와 동일하게 사용할 수 있습니다.

다만 Play,Stop()등 동작 관련 API들을 추가로 지원합니다.

기본적인 사용방법은 ImageVIew와 동일합니다.

다만 Play,Stop과 같은 기능을 추가로 수행할수 있으며 이를 위해서 약간의 cs 코드 수정이 필요합니다.

이번 튜토리얼에선 Play,Stop을 위해 버튼을 추가하였습니다.

<ImageView x:Name="imgView"
               ResourceUrl="*Resource*/images/dog-anim.gif"
               Size2D="400, 400"/>
<Button Text="Play" Size="100, 100" Clicked="Play_Clicked"/>
<Button Text="Stop" Size="100, 100" Clicked="Stop_Clicked"/>

위처럼 Clicked에 맞는 함수들이 cs파일에 자동으로 생성되고,

해당 파일에서 아래처럼 Play, Stop등의 추가 기능을 사용하실 수 있습니다.

private void Play_Clicked(object sender, Tizen.NUI.Components.ClickedEventArgs e)
{
  imgView.Play();
}

private void Stop_Clicked(object sender, Tizen.NUI.Components.ClickedEventArgs e)
{
  imgView.Stop();
}

두번째로 Lottie 사용법에 대해 설명드리겠습니다.

Lottie는 gif와 비슷하지만 좀 더 쉽게 high-quality 애니메이션을 사용할 수 있도록 하는 포맷입니다.

해당 포맷은 ImagView에서 바로 사용할 수는 없지만 대신 별도의 View인 LottieAnimationView를 통해 사용할 수 있습니다.

사용방법은 ImageView와 비슷하며 gif처럼 animated 관련된 몇가지 기능이 추가로 제공됩니다.

Lottie에 대한 설명과 세부적인 내용은 아래 문서를 참고 부탁드립니다.

https://docs.tizen.org/application/dotnet/guides/nui/nui-components/LottieAnimationView/

<LottieAnimationView x:Name="lottieView"
               ResourceUrl="*Resource*/images/loading_circle.json"
               Size2D="600, 600"
               LoopCount="-1"
               BackgroundColor="#FFFFFF"/>

Play,Stop 사용법은 gif와 동일합니다.

실제 동작 화면은 실습을 통해 보여드리겠습니다.

4. Image View Event

Image Load가 끝난 시점에 필요한 동작이 있는 경우를 위해

Image View에서는 Resource Ready Event를 지원합니다.

이 기능의 경우도 XAML 내에서 쉽게 사용할 수 있습니다.

아래 코드처럼 ResourceReady를 입력하면 자동으로 Callback 함수가 생성되고

해당 Callback에 원하는 동작을 구현하면 됩니다.

<ImageView x:Name="imgView"
               ResourceUrl="*Resource*/images/Poster_1.png"
               Size2D="600, 600"
               PixelArea="0.0, 0.0, 1.0, 1.0"
               FittingMode="ShrinkToFit"
               ResourceReady="imgView_ResourceReady"
               BackgroundColor="#000011"/>

아래처럼 XAML내에서 선언한 Callback이 Scene1Page.xaml.cs에 자동으로 생성됩니다.

이 Callback에서 원하는 동작을 추가로 구현할 수 있습니다.

만약 ImageView의 다른 기능을 이 시점에 사용하고 싶으면 XAML 내에서 지정한 Name을 여기서 변수명으로 사용할 수 있습니다.

예를 들어 이미지 로드가 끝난 시점에 이미지의 위치를 이동하고 싶다면 아래 코드처럼 구현할 수 있습니다.

private void imgView_ResourceReady(object sender, ImageView.ResourceReadyEventArgs e)
{
  imgView.Position2D = new Position2D(400, 0);
}

/// ScrollEventArgs
public class ScrollEventArgs : EventArgs
{
   /// Position of ContentContainer
   public Position Position;
   /// Position of Scroll
   public Position ScrollPosition;
}

namespace Hackerton.NUI.Lesson5
{
    class MyListItem
    {
        readonly string resourcePath = Tizen.Applications.Application.Current.DirectoryInfo.Resource + "/images/flags/";
        readonly Color[] colors = new Color[] { new Color("#96ceb4"), new Color("#ffeead"), new Color("#ff6f69"), new Color("#ffcc5c"), new Color("#88d8b0"),
                                                new Color("#4a4e4d"), new Color("#0e9aa7"), new Color("#3da4ab"), new Color("#f6cd61"), new Color("#fe8a71")  };
        public int Index { get; set; }
        public string Name { get => Index + "th Item"; }
        public string FlagIconPath { get => resourcePath + "flag_" + (Index % 50) + ".png"; }
        public Color BgColor { get => colors[Index % 10]; }

        public MyListItem(int index)
        {
            Index = index;
        }
    }

    class MyListSource
    {
        public List<MyListItem> Source { get; private set; } = null;
        public MyListSource(int count)
        {
            Source = new List<MyListItem>();
            for (int i = 0; i < count; i++)
            {
                Source.Add(new MyListItem(i));
            }
        }
    }
}
