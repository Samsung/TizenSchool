I. Introduction

2. How data binding works

Binding basics

Data binding brings easy way to update UI of application. It connects two properties of objects so that change in one object causes change in the other. Data bindings can be defined in code or in XAML. This mechanism is very important in MVVM pattern, because it allows to completely separate view from business logic.
In data binding we can distinguish binding source  and binding target . Target property should be registered by creating BindableProperty , so it can listen for any changes in source property.

For example Label has bindable property Text  and we have object Animal  with string property Name . So Text property is Target  and Name  is Source .
In data binding we must inform our target properties about changes in sources. Framework provides us with INotifyPropertyChanged  interface. This interface contains event that should be fired, when we want to update source property.

Setting binding

Binding can be defined in C# code:

var sourceObject = new SourceData();
var myBinding = new Binding("MyProperty", BindingMode.OneWay)
{
    Source = sourceObject
};
MyLabel.SetBinding(Label.TextProperty, myBinding);

Or in XAML:

<Label>
    <Label.Text>
        <Binding Path="MyProperty">
            <Binding.Source>
                <local:SourceData />
            </Binding.Source>
        </Binding>
    </Label.Text>
</Label>

Setting explicitly source can be ommited by setting data context . In Xamarin.Forms we can set it by using property BindingContext.  Data context is object that contains Source property. It can be simple type or class (In fact it doesn’t have to be CLR object. XMLs, dynamic objects or entities also can be sources for data binding).  After setting binding context, we have access to its public properties. Binding context is inherited from parent, so if we set binding context for page it will be accessible for whole content of that page.

Types of binding

As you can see in examples, while creating binding we used property named Mode of type BindingMode. We used default value, but there are three options, that we can use. Those values indicate direction of data flow. Sometimes we want to let user change some data and propagate it to source object. 2. View-to-view binding

To make view-to-view binding firstly we must define source and target. In our case we want to take values from sliders and rotate label by those values. Then a target is Rotation property of label and a source is value of slider. We have three axes, so there are three sliders. As you can see each slider has defined x:Name, what is needed to create view-to-view binding.

Add following propreties to our label.

<ContentPage.Content>
    <Grid>
        <Grid.RowDefinitions>
            <RowDefinition Height="*" />
            <RowDefinition Height="Auto" />
            <RowDefinition Height="250" />
            <RowDefinition Height="Auto" />
        </Grid.RowDefinitions>

        <Label Text="Hello Tizen!"
                Grid.Row="0"
                RotationX="{Binding Source={x:Reference Name='XSlider'}, Path=Value}"
                RotationY="{Binding Source={x:Reference Name='YSlider'}, Path=Value}"
                Rotation="{Binding Source={x:Reference Name='ZSlider'}, Path=Value}"
                HorizontalOptions="Center"
                VerticalOptions="Center" />

        <StackLayout Grid.Row="1"

This notation says that value for rotation will be taken from other property by binding mechanism. Our source is slider object and its Value property.

After this modification we could run our application and test it. Label should rotate now.

3. Creating context class for page

In order to create different types of binding we must create data context for our view. So add new class named ViewModel .cs :

As I mentioned in introducion this class must implement INotifyPropertyChanged interface from System.ComponentModel namespace.

using System.ComponentModel;
using System.Runtime.CompilerServices;

namespace BindingTutorial
{
    class ViewModel : INotifyPropertyChanged
    {
        public event PropertyChangedEventHandler PropertyChanged;

        protected virtual void OnPropertyChanged([CallerMemberName] string propertyName = null)
        {
            PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
        }
    }
}

Now if we want to update view, we have to call method  OnPropertyChanged.

4. One-way binding

In this section we will fill labels that display numeric values of rotation angles.

Firstly we need to set Binding the ViewModel. In MainPage.xaml add following lines:

<ContentPage xmlns="http://xamarin.com/schemas/2014/forms"
             xmlns:x="http://schemas.microsoft.com/winfx/2009/xaml"
             x:Class="BindingTutorial.MainPage"
             xmlns:main="clr-namespace:BindingTutorial;assembly=BindingTutorial"
             BackgroundColor="White">
    <ContentPage.BindingContext>
        <main:ViewModel />
    </ContentPage.BindingContext>

    <ContentPage.Content>
        <Grid>

Now we have created new  instance of Context class and set it as binding context.

Hence, we are ready to add properties that will be our sources. Open ViewModel.cs file and add three integer properties:

But as I mentioned before we need to invoke PropertyChanged event when property is changed. So let’s do this:

namespace BindingTutorial
{
    class ViewModel : INotifyPropertyChanged
    {
        private int _x;
        private int _y;
        private int _z;

        public int X
        {
            get => _x;
            set
            {
                _x = value;
                OnPropertyChanged();
            }
        }

        public int Y
        {
            get => _y;
            set
            {
                _y = value;
                OnPropertyChanged();
            }
        }

        public int Z
        {
            get => _z;
            set
            {
                _z = value;
                OnPropertyChanged();
            }
        }

As you can see, we had to add private fields that hold actual values of properties.

Now, when we have source properties, we are ready to create actual binding. Go back to MainPage.xaml and change Text property of each label as follows:

<StackLayout Grid.Row="1"
                Orientation="Horizontal"
                HorizontalOptions="Center">
    <Label Text="{Binding X, StringFormat='X: {0}'}"
            Margin="50, 0, 50, 0" />
    <Label Text="{Binding Y, StringFormat='Y: {0}'}"
            Margin="50, 0, 50, 0" />
    <Label Text="{Binding Z, StringFormat='Z: {0}'}"
            Margin="50, 0, 50, 0" />
</StackLayout>

We are almost done. We set up binding for displaying angle of rotation, but we don’t update it in any way. We need to create another binding with our X, Y, Z properties and values of rotations from sliders. Just change Value properties of sliders:

<Slider x:Name="XSlider"
        Minimum="0"
        Maximum="360"
        Value="{Binding X, Mode=OneWayToSource}"
        Margin="50, 0, 50, 0" />
<Slider x:Name="YSlider"
        Minimum="0"
        Maximum="360"
        Value="{Binding Y, Mode=OneWayToSource}"
        Margin="50, 0, 50, 0" />
<Slider x:Name="ZSlider"
        Minimum="0"
        Maximum="360"
        Value="{Binding Z, Mode=OneWayToSource}"
        Margin="50, 0, 50, 0" />
</StackLayout>

Please note that we had to change Mode of binding to OneWayToSource , because we want to update Source not Target.

Now labels should display angles of rotations.

5. Two-way binding

In this final section we will create action for Reset button.

Firstly we need to create method that will be executed on button click. Open ViewModel.cs file and add new method:

        _z = value;
        OnPropertyChanged();

}

private void ResetValues()
{
    X = Y = Z = 0;
}

Now we need property of type Command from Xamarin.Forms namespace. Command is a special type that helps with data binding and events. Some classes supports using commands instead of events. That allows us to move UI logic to context classes rather than handling them in code behind.

Commanding is supported by following Xamarin.Forms classes:

Hence, we need to create Command property in our ViewModel class and create instance in class constructor. In parameter we pass our new method.

public Command ResetCommand { get; set; }

private void ResetValues()
{
    X = Y = Z = 0;
}

public ViewModel()
{
    ResetCommand = new Command(ResetValues);
}

Now in MainPage.xaml we could add logic to button.

 <Button Text="Reset"
         Command="{Binding ResetCommand}"
         Grid.Row="3"
         Margin="50, 10, 50, 10" />

Unfortunately this is not enough. Now we want to change sliders values from UI and from code. We need to change binding mode to TwoWay:

<Slider x:Name="XSlider"
        Minimum="0"
        Maximum="360"
        Value="{Binding X, Mode=TwoWay}"
        Margin="50, 0, 50, 0" />
<Slider x:Name="YSlider"
        Minimum="0"
        Maximum="360"
        Value="{Binding Y, Mode=TwoWay}"
        Margin="50, 0, 50, 0" />
<Slider x:Name="ZSlider"
        Minimum="0"
        Maximum="360"
        Value="{Binding Z, Mode=TwoWay}"
        Margin="50, 0, 50, 0" />

Ok, now button should set rotations to 0.
